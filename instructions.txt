I want to create an electron APP, that is targeted towards Human resources managers. The app in general is a parser, from a database that has a lost of resumes in pdf and word format. The recruiter sets a few keywords that they want in the CV, for example it can be a certification or tech stack and etc. Now, I want there to be two categories of keywords, one is mandatory keywords and the other is extra keywords that depends on the recruiter. All the resumes that don't have the mandatory keywords are rejected and if the CV has emails of the applicants then they're sent an email informing them that they're rejected. The accepted CVs are put in two categories, ones that have just the mandatory keywords and ones that have at least one of the optional keyword. The accepted CV's are then sent to a local LLM, which reads the CVs and in the context of a prompt that the recruiter gives, it gives out a list of names, their CVs and a small paragraph about why the LLM reasoned that they're a good fit. Now, I will polish the tiny leftover details in a bit, but for now I want this app to follow the following technical guidelines: 
1. The app must be able to accept local data as well as cloud data for the parsing script. 
2. The LLM is going to be run locally, but it might be possible that the llm is running locally on a different physical device, I want to be able to configure the app ti run local LLM on the system, or local llm from another system. Preferrably using ollama. 
3. The app must be scalable, if the app is installed on a server that also runs other apps and programs, it should be compatible with the resource managing softwares and should also be able to run in the background. 

All the possible errors must have an error code and location of error whether its the llm or any other part of the app that fails. I want a documentation as well that describes the app and has detailed info about the error codes.
